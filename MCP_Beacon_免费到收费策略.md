# 🆓→💰 MCP Beacon 免费积累 → 逐步收费方案

> 版本：V1.0 | 日期：2026-06-01 | 决策：Ari 拍板"先免费跑"

---

## 一、核心逻辑

```
免费 MVP → 积累用户 → 采集数据 → 迭代优化 → 逐步收费
   ↑                                              |
   └──────── 数据反馈驱动 ────────────────────────┘
```

| 阶段 | 目标 | 关键指标 | 时长 |
|------|------|----------|:---:|
| **Phase 0: 免费 MVP** | 产品可用、有人用 | 注册用户数、SDK 安装量 | W1-12 |
| **Phase 1: 用户积累** | 100+ 活跃开发者 | DAU、留存率、NPS | 上线后 1-3 月 |
| **Phase 2: 数据迭代** | 根据真实使用打磨产品 | 功能使用率、痛点反馈 | 持续 |
| **Phase 3: 渐进收费** | 5-10% 付费转化 | MRR、转化率、流失率 | 指标触发后 |

## 二、Phase 0-1：免费 MVP（W1-12 → 上线后 3 个月）

### 2.1 免费套餐设计

| 项目 | 免费版 |
|------|--------|
| Agent 数量 | **3 个**（够个人开发者用） |
| 数据保留 | **30 天** |
| 核心功能 | 全部开放（监控/仪表盘/成本追踪/告警） |
| 异常检测 | ✅ 全部开放 |
| 团队协作 | ❌ （Pro 功能） |
| 导出数据 | CSV 导出 ✅ |
| 支持 | 社区（GitHub Issues） |

> **设计原则**：免费版要足够好用到让人"愿意付费升级"，而不是"被迫付费解锁"。参考 Linear/Figma 模型。

### 2.2 用户获取渠道（Ari 可访问）

| 渠道 | 方式 | 预期效果 |
|------|------|:---:|
| **掘金** | 技术文章：MCP Beacon 接入教程 | 🥇 国内 AI 开发者浓度最高 |
| **知乎** | 回答 MCP/Agent 监控相关话题 | 🥈 长尾流量 |
| **GitHub** | README 优化 + awesome-mcp 提交 | 🥇 开发者自然发现 |
| **npm** | 包名 `mcp-beacon` + 关键词优化 | 🥈 搜索流量 |
| **少数派** | 效率工具推荐 | 🥉 非技术用户 |
| **ProductHunt** | 上线日提交（Ari 无需参与） | 🥈 海外开发者 |

> ⚠️ 砍掉的渠道：Twitter/X（Ari 无法访问）、Reddit、HackerNews

### 2.3 用户数据采集（关键！）

**原则：匿名 + 合规 + 最小化**

| 采集维度 | 具体数据 | 用途 |
|----------|----------|------|
| **接入数据** | SDK 安装数、首次接入成功率 | 衡量 SDK 易用性 |
| **使用行为** | 日均调用量、功能使用频率、页面停留 | 发现高频/低频功能 |
| **留存** | D1/D7/D30 留存率 | 核心健康指标 |
| **流失信号** | 7天未登录、SDK 未上报数据 | 提前干预 |
| **NPS** | 接入后 7 天自动弹 NPS 问卷 | 量化满意度 |

**工具选择（免费优先）**：

| 工具 | 用途 | 费用 |
|------|------|:---:|
| **Supabase 内置** | 用户行为事件存储 | $0（已有） |
| **自建轻量埋点** | SDK 端 + Dashboard 端事件采集 | $0 |
| **GitHub Issues** | 用户反馈 + Feature Request | $0 |
| **NPS 弹窗** | 自建组件，7 天触发 | $0 |

> ❌ 不用 Google Analytics（Ari 无法访问）
> ❌ 不用 Sentry（等收费后再加）

## 三、Phase 2：数据驱动迭代

### 3.1 反馈闭环

```
用户行为数据 ──→ 每周 Review ──→ 优先级排序 ──→ 开发迭代
     │                                          │
     └────────── 效果验证 ←──────────────────────┘
```

### 3.2 关键决策数据

| 如果看到... | 则应该... |
|-------------|-----------|
| SDK 安装后 50%+ 未成功上报 | 优先优化 onboarding 流程 |
| 仪表盘访问 > 告警使用 | 告警功能可能需要简化 |
| 成本面板 DAU 最高 | 成本是核心痛点，收费后主打 |
| D7 留存 < 30% | 产品价值不够，暂不收费 |
| NPS < 30 | 先解决核心体验问题 |
| NPS > 50 + D30 > 40% | ✅ **启动收费的信号** |

### 3.3 迭代节奏

| 周期 | 动作 |
|------|------|
| **每周** | 查看 Supabase 用户行为数据 → 输出 3 条洞察 |
| **每两周** | 小版本迭代（修 bug + 体验优化） |
| **每月** | Ari Review 数据 → 决定是否进入收费阶段 |

## 四、Phase 3：渐进收费

### 4.1 收费触发条件（AND 逻辑）

| 条件 | 阈值 |
|------|:---:|
| 注册用户 | > 200 |
| 周活跃用户 (WAU) | > 50 |
| D30 留存率 | > 40% |
| NPS | > 40 |
| 用户明确要求的功能 | 至少 3 个"愿意付费"的 Issue |

> 达到以上 **全部条件** 后，启动收费计划。

### 4.2 收费节奏（阶梯式，不搞一刀切）

```
第1个月：功能收费，不搞强制付费
  → 新增 Pro 功能（团队协作/更长数据保留/高级告警）
  → 现有免费用户一切不变
  → 早期用户给 50% 永久折扣（感谢信 + 折扣码）

第2个月：观察转化
  → 免费版保留（不砍功能）
  → 新用户默认 Free，引导升级 Pro

第3个月：定价优化
  → 根据转化数据调整定价
```

### 4.3 收费套餐（渐进版）

| 套餐 | 价格 | 与免费版差异 | 
|------|:---:|------|
| **Free** | $0 | 3 Agent + 30天数据 + 基础告警 |
| **Pro** | **$15/月** | 10 Agent + 90天数据 + 高级告警 + Slack 通知 |
| **Team** | **$39/月** | 无限 Agent + 无限数据 + 团队协作 + API |

> 💡 Pro 定 $15 而非原来的 $19——低价降低付费摩擦，等用户基数大了再提价。

### 4.4 支付方案（Ari 在中国可用）

| 方案 | 推荐度 | 理由 |
|------|:---:|------|
| **Lemon Squeezy** | 🥇 | 个人可注册、全球税务自动处理、API 好对接 |
| **先不做支付** | ✅ 当前 | MVP 阶段不需要，等收费触发后再接 |

> 收费触发后，Ari 只需注册 Lemon Squeezy（30分钟），然后 AI Coder 对接 API。
