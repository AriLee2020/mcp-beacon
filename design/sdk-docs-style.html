Introduction to MCP Beacon

MCP Beacon is an observability platform for Model Context Protocol (MCP) based AI agent systems. It provides real-time monitoring, cost intelligence, and alerting — all through a transparent proxy that requires zero code changes to your agents.

New to MCP? The Model Context Protocol is an open standard for AI agent tool calling. Read the MCP specification first if you're unfamiliar.

How It Works

Beacon sits between your MCP client and your MCP servers. It forwards all traffic while collecting telemetry — no instrumentation required.

Point your MCP client to Beacon

Replace your MCP server URL with Beacon's proxy endpoint. That's it — one config change.

Beacon proxies traffic transparently

All MCP calls flow through Beacon. It records timing, tokens, tools used, and errors without modifying requests.

View dashboards and get alerts

Real-time metrics appear in your dashboard. Set up budget thresholds and anomaly alerts.

Quick Start

Install the Beacon SDK and update your MCP client config:

# Install the SDK
npm install @mcp-beacon/sdk

# Or with Python
pip install mcp-beacon
// mcp-config.json — before
{
  "mcpServers": {
    "my-tools": {
      "url": "https://my-mcp-server.com"
    }
  }
}

// mcp-config.json — after (one line changed)
{
  "mcpServers": {
    "my-tools": {
      "url": "https://beacon.mcpbeacon.dev/proxy/my-tools",
      "headers": {
        "X-Beacon-API-Key": "bcn_your_api_key"
      }
    }
  }
}

Python SDK Usage

For Python-based agents, use the mcp-beacon package:

from mcp_beacon import BeaconClient

# Initialize the client
beacon = BeaconClient(
    api_key="bcn_your_api_key",
    project="my-agent-project"
)

# Wrap your MCP server calls
async def call_tool(tool_name, arguments):
    async with beacon.trace(tool_name) as span:
        result = await actual_mcp_call(tool_name, arguments)
        span.set_metadata({"tokens": result.usage.total_tokens})
        return result
