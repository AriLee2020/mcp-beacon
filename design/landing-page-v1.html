FAQ

Common questions

What is MCP and why do I need monitoring for it?
The Model Context Protocol (MCP) is the standard for AI agent tool calling. As your agent fleet grows, you need visibility into what each agent is doing, how much it costs, and whether it's performing correctly. Beacon provides that visibility.
How does Beacon integrate with my existing agents?
Beacon acts as a transparent proxy. You point your MCP client to Beacon's endpoint instead of directly to your MCP servers. Beacon forwards all traffic while collecting telemetry — zero code changes to your agents.
Is my data secure? Can I self-host?
Yes and yes. All data is encrypted in transit and at rest. Enterprise plans include on-premise deployment. Beacon never stores your actual agent prompts or responses — only metadata and metrics.
What AI providers do you support?
Beacon is provider-agnostic. Since it sits at the MCP protocol layer, it works with any MCP-compatible setup regardless of the underlying LLM provider (OpenAI, Anthropic, open-source models, etc.).
